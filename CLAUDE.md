# Gapfill（溜缝儿）

AI 开发者的通用工具箱。

## 项目规则

### 命令执行
- 一次执行一个命令，不要使用 `&&`、`;`、`||` 链式组合
- 先 `cd` 切换目录，再在单独的步骤中执行后续命令
- 这确保每个命令都能被 Claude Code 权限系统单独匹配

### 编码原则
- 先阅读现有代码和文件，再提出修改建议
- 保持修改最小化和聚焦
- 不要添加超出需求的功能
- 所有文档、注释、commit message 使用中文

### 分支工作流
- `main` 是受保护分支，不允许直接 push
- 所有新功能在 `feature/<功能名>` 分支上开发
- 测试通过后合并到 `main`，通过 Pull Request 推送
