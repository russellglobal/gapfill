## 项目概述

{{project_name}} - AI 辅助开发项目。

## 开发原则

- 保持改动最小化，聚焦于需求本身
- 不要添加超出需求的功能
- 提交前验证，不要跳过测试
- 遵循现有模式和约定
- 不要使用复合命令（`&&`、`;`、`||`）——每个命令单独执行，以便逐一审批
- 所有文档、注释、提交信息使用中文（对外公开项目除外）

## 分支工作流

（适用于使用托管平台的项目）

- `main` 为受保护分支，不允许直接推送
- 所有新功能在 `feature/<名称>` 分支上开发
- 测试通过后通过 Pull Request 合并到 `main`

## 常用命令

每条命令单独执行，不要链式调用。

- 查看状态: `git status`
- 查看差异: `git diff`
- 查看日志: `git log --oneline -10`

## 安全规则

- 不要在代码或注释中暴露密钥、API 凭据
- 不要在没有用户明确确认的情况下执行破坏性命令（`rm -rf`、`git reset --hard`）
- 不要跳过 git hooks（`--no-verify`、`--no-gpg-sign`）
