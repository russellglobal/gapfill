# gapfill 哲学（溜缝儿）

## 定位：填补 AI 编码助手的"管理真空"

Claude Code 擅长写代码，但不负责管理项目。gapfill 补上这块空白——权限预设、文档骨架、git 工作流、提交前审查。它不写一行业务代码，却让 Claude Code 在每个项目里都能安全、高效地工作。

**一句话：gapfill 是 Claude Code 的基础设施，不是竞争对手。**

## 核心原则

### 1. 零 token 消耗

gapfill 的所有操作——初始化、审查、扫描——都在本地执行，不调用任何 LLM API，不发送任何代码或配置到外部。

**为什么重要：** 用户的代码和权限配置属于敏感信息，不应该为了"生成一个项目骨架"就把它们送出去。零 token 也意味着零成本、零延迟、零网络依赖。

### 2. 确定性

同样的输入，同样的输出。没有随机性，没有"AI 幻觉"，没有不可复现的结果。gapfill 是一个 deterministic tool——你今天跑和明年跑，结果一致。

**为什么重要：** 项目初始化是基础设施行为，不应该有"创意"。创意留给 Claude，确定性交给 gapfill。

### 3. 安全优先

- 权限模板只包含安全操作（read/git/build），不含 `Write(/**)` 等高危规则
- 绝不覆盖用户已有文件（CLAUDE.md、settings.local.json）
- 已有项目不自动 commit
- 危险操作需要用户确认

**为什么重要：** 权限配置是安全边界，一旦放宽就很难收回。默认安全比默认方便更重要。

### 4. 极简主义

- 零外部依赖，只要 Python 3.8+ 和 git
- 没有配置文件，没有数据库，没有后台服务
- 一个命令做完就走，不驻留

**为什么重要：** 管理工具本身不应该成为需要被管理的负担。

### 5. 尊重用户主权

- 用户已有配置时，询问而非覆盖
- 非交互模式默认保守（跳过而非强制）
- 所有输出可读、可审查、可拒绝
- git 历史不可变——不在已有仓库里强制提交

**为什么重要：** 工具应该增强用户控制，而不是替代用户决策。

## 名字的含义

**gapfill** —— 填补空白。填补 Claude Code 能力与实际项目管理需求之间的缝隙。

**溜缝儿** —— 中文口语，指填补墙面或物体的细小裂缝。引申义：修补那些"不算大问题但总让人不舒服"的细节。不是大修，是精细化打磨。

## 不是什么

- **不是** Claude Code 的替代品——是它的前置准备和后置保障
- **不是** 项目脚手架（如 create-react-app）——不生成业务代码或框架模板
- **不是** 配置管理器（如 Ansible）——不做系统级配置，只管 Claude Code 项目
- **不是** AI 工具——不调用任何模型，纯本地确定性执行

## 价值主张

| 维度 | 没有 gapfill | 有 gapfill |
|------|-------------|-----------|
| 新项目启动 | ~15 分钟手动配置 | 1 个命令，~5 秒 |
| Token 消耗 | 初始化可能消耗 token（如果用 AI 生成） | 0 |
| 安全风险 | 手动配置可能遗漏或误配 | 模板统一，可审计 |
| 权限管理 | 每个命令点"允许" | 预置权限，减少 ~80% 确认 |
| 提交前检查 | 凭经验或忘记检查 | 7 项自动检查 |
| 多项目一致性 | 每个项目配置不同 | 统一模板，一致性保证 |

## 产品规划

### 不做"企业级"，做不可替代

五个专家视角的共识只有一个："企业级"不是 gapfill 的方向。不是因为这个需求不重要，而是因为切入点不对。

- **产品策略**："CLI 工具采购"这个企业市场不存在。应该做 AI 辅助开发的默认配置层——像 husky/prettier 一样成为生态基础设施
- **安全合规**：安全团队买的是可证明的合规，不是安全默认值。gapfill 的定位是"开发者工作站合规层"——被企业的 AI 使用政策引用，而非作为管理后台被采购
- **开发者体验**：最好的工具对一件事极度固执。"企业级"会毁掉 gapfill 最稀缺的品质：零 token、确定性、用户主权
- **中国市场**："企业级"在中国开发者社区约等于"贵、重、难用"。"溜缝儿"是最大的差异化——接地气、不端着、符合"别整虚的，能跑就行"的实用主义
- **生态防御**：被平台吸收的风险存在，但跨平台治理层就是护城河。做 AI 编码工具的 ESLint——不 glamorous，但团队一扩大就离不开

### 三步路径

1. **先做透开发者体验**——更多模板（Next.js、FastAPI、Go、Rust），让 `gapfill init` 成为从 `mkdir` 到可用 AI 编码的最快路径
2. **抽象到跨平台**——不绑定 Claude Code，支持 Cursor、Copilot 等，做 AI 编码工具的通用治理层
3. **企业渗透靠自下而上**——开发者装了，老板管不住。不是卖许可证，是被引用为基线基础设施

**一句话：不追求企业级，追求不可替代。当每个 AI 编码用户都用 gapfill 时，企业级自然就来了。**
